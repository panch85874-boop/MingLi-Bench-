# CLAUDE.md — 專案協作偏好

## 互動方式（重要）

- **每次需要使用者做決定或回答問題時，一律使用「可點選選項」的方式提問**（`AskUserQuestion` 工具），不要用需要打很多字才能回答的開放式問題。
- 把選項設計成可直接點選（必要時用多選），讓使用者用最少的輸入就能回答。
- 只有在真的無法用選項表達、必須自由輸入時，才用開放式問題。
- 使用者偏好高效率溝通：先給結論與選項，少長篇大論。

### 已拍板的關鍵決策
- 自助優先、**不靠真人**：硬體問題導「改用店內另外兩台機器（三台、間隔約 1 公尺、未完成不扣款）」。
- **不是 AI Agent / 不裝 LLM**：落地主機只跑 ASR + 句子向量比對 + TTS，輸出只能是固定句庫的某一句，不自由生成。
- 比對邏輯：**模糊語意比對、永遠回最接近的一句**（無「沒資料」回覆）；只有完全沒聽到語音才請客人再說一次。
- 退款過渡（決策 1B）：付款後沒印好 → 只導重印不退費；扣款爭議 → 致歉＋保留交易紀錄＋告知線上申請即將開放。退款自助入口（D-A）為 P0 待開發。
